# 项目协作规范

## 优化计划维护

- 项目优化路线以 `docs/claude_quant_optimization_plan.md` 为单一事实来源。
- 每次开始较大的优化前，先查看该文件的“当前进度”和“下一步优化动作”。
- 每次完成优化、修复、调整优先级或发现新风险后，同步更新该文件中的进度，避免只依赖对话记忆。
- 提交代码前检查本文件是否需要更新。

## 排查 Web 端页面问题

- **重点关注服务端接口交互**：排查 web 端页面问题时，优先检查 API 请求/响应、SSE 数据流、错误状态码等与后端的交互，而不是 UI 高亮、样式、CSS 类等视觉问题。
- **使用 Chrome DevTools MCP 调试**：不要用 Playwright 写测试的方式进行排查，直接使用 Chrome DevTools MCP 进行实时调试。
